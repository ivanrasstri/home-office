# Мануал: удалённая разработка через Claude Code on the web

Этот документ простыми словами объясняет, что настроено в репозитории для работы
с **Claude Code on the web** (кодинг в облаке, без локального компьютера), и что
вообще при этом происходит.

---

## 1. Что такое «кодить удалённо»

Обычно Claude Code работает у тебя на компьютере. **Claude Code on the web** —
это то же самое, но в облаке:

- Открываешь сессию в браузере / приложении.
- В облаке поднимается **временный контейнер** (мини-сервер).
- В него **с нуля клонируется** этот репозиторий.
- Ты ставишь задачу — Claude правит код прямо там.

> ⚠️ Контейнер **эфемерный**: после простоя он удаляется. Всё, что не
> закоммичено и не запушено в GitHub, — теряется. Поэтому изменения всегда
> попадают в git (ветку и PR).

---

## 2. Что было настроено

Добавлены три вещи. Ниже — что это за файлы и зачем.

### 2.1. SessionStart-хук — «подготовить окружение при старте»

**Файлы:** `.claude/hooks/session-start.sh` + `.claude/settings.json`

Когда стартует новая облачная сессия, контейнер чистый — в нём ещё не
установлены Python-библиотеки проекта (`requests`, `PyYAML`, `anthropic` и т.д.).
Хук — это скрипт, который **запускается автоматически при старте сессии** и всё
доставляет:

```bash
# суть скрипта:
pip install -r requirements.txt          # ставит зависимости проекта
export PYTHONPATH=...                     # чтобы работала команда python -m jobbot
```

Благодаря этому сразу после старта можно, например, выполнить:

```bash
python -m jobbot collect      # собрать подборку вакансий
python -m jobbot apply --ids 123,456
```

…и не получить ошибку «модуль не найден».

Особенности:
- Скрипт работает **только в облаке** — он проверяет переменную
  `CLAUDE_CODE_REMOTE`. На твоём личном компьютере он ничего не делает (там ты
  сам управляешь своим окружением / venv).
- `.claude/settings.json` просто **регистрирует** этот скрипт как хук, чтобы
  Claude Code знал, что его надо запускать.

**Режим: синхронный.** Сессия стартует только после того, как зависимости
установлены. Плюс — к началу работы всё готово, нет «гонок». Минус — старт чуть
дольше. Можно переключить на асинхронный (быстрее старт, но Claude может начать
работать раньше, чем доустановятся пакеты).

### 2.2. MCP — «подключить внешние инструменты к Claude»

**Файл:** `.mcp.json`

**MCP (Model Context Protocol)** — стандарт, через который Claude получает доступ
к внешним инструментам и сервисам: открывать веб-страницы, работать с GitHub,
Gmail, базами данных и т.п. Без MCP Claude видит только то, что есть в репозитории.

Сейчас подключён один сервер — **`fetch`**:

```json
{
  "mcpServers": {
    "fetch": { "command": "uvx", "args": ["mcp-server-fetch"] }
  }
}
```

Он даёт Claude возможность **открывать и читать веб-страницы**. Для этого проекта
это полезно: сборщики вакансий живут в `jobbot/sources/` (HeadHunter, RemoteOK,
We Work Remotely и др.), и при их отладке удобно, чтобы Claude мог сам глянуть,
что реально отдаёт сайт или RSS-лента.

Важно:
- Это **project-scoped** сервер (лежит в репозитории, общий для всех сессий
  проекта). При первом запуске Claude Code **спросит подтверждение** — это
  защита от запуска чужого кода. Подтвердить можно командой `/mcp`.
- Секретов в файле нет — `fetch` не требует ключей.

## 3. Как этим пользоваться

1. **Открыть веб-сессию** на этом репозитории — окружение поднимется само
   (хук поставит зависимости).
2. **Поставить задачу** Claude обычным текстом.
3. Claude правит код, **коммитит и пушит** в ветку → обновляется PR.
4. Когда всё устраивает — **смёрджить PR в `main`**. После этого настройки
   (хук + MCP) станут действовать во **всех будущих** сессиях.

## 4. Как расширять

### Добавить ещё MCP-сервер
Допиши его в `.mcp.json`. Форматы серверов и опции — в доке:
https://code.claude.com/docs/en/mcp

Пример с удалённым HTTP-сервером GitHub (авторизация через `/mcp`, без секретов
в файле):

```json
{
  "mcpServers": {
    "fetch":  { "command": "uvx", "args": ["mcp-server-fetch"] },
    "github": { "type": "http", "url": "https://api.githubcopilot.com/mcp/" }
  }
}
```

### Изменить, что ставится при старте
Редактируй `.claude/hooks/session-start.sh` (например, добавь установку
дополнительных инструментов или экспорт переменных через `$CLAUDE_ENV_FILE`).

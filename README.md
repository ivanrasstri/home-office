# home-office — бот для поиска работы

Персональный бот, который ищет вакансии под твой профиль (продюсер креативной
студии / креативный проектный менеджер), оценивает их, ведёт список уже
просмотренных и готовит отклики — сопроводительное письмо и адаптированное под
вакансию резюме. Запускается по расписанию в **GitHub Actions** (свой сервер не нужен).

## Что он делает

1. **Собирает вакансии** из нескольких источников:
   - **HeadHunter** (api.hh.ru) — официальный API, основной источник для СНГ/Узбекистана.
   - **RemoteOK** и **We Work Remotely** — удалёнка (англоязычные борды).
   - **Telegram** — публичные каналы с вакансиями (через `t.me/s/<channel>`).
   - **LinkedIn** — гостевой поиск (best-effort: часто упирается в лимиты с IP
     GitHub Actions; при ошибке просто пропускается).
2. **Оценивает** каждую вакансию баллом 0–100 по ключевым словам, локации и формату.
3. **Отсеивает** уже виденные (дедуп через `data/seen.json`) и слишком слабые по баллу.
4. **Формирует отчёт** `reports/<дата>.md` (+ `reports/latest.md`) с топ-вакансиями.
5. **Готовит отклики** для топ-N вакансий через Claude → `applications/` (нужен `ANTHROPIC_API_KEY`).
6. **Коммитит результаты** обратно в репозиторий — отчёты и отклики копятся в истории.

## Быстрый старт (локально)

```bash
pip install -r requirements.txt
cp .env.example .env           # впиши ANTHROPIC_API_KEY (необязательно)
# подгрузить переменные окружения и запустить:
set -a && . ./.env && set +a
python -m jobbot
```

Без `ANTHROPIC_API_KEY` бот всё равно соберёт и оценит вакансии и сделает отчёт —
просто не будет генерировать письма/резюме.

## Настройка под себя

- **`config/profile.yaml`** — главный файл. Роль, поисковые запросы, ключевые
  слова для скоринга, стоп-слова, локация, параметры hh. Держи его актуальным.
- **`config/sources.yaml`** — какие источники включены; список Telegram-каналов и
  RSS-лент We Work Remotely. Подставь свои каналы.
- **`resume/resume.md`** — твоё базовое резюме. На его основе Claude делает
  адаптированные версии. **Заполни реальными данными** — бот не выдумывает факты.

## Запуск в GitHub Actions

Workflow: `.github/workflows/job-search.yml` — по будням в 06:00 UTC (~11:00 Ташкент)
и вручную (вкладка **Actions → Job Search Bot → Run workflow**).

Настройка:
1. Положи `ANTHROPIC_API_KEY` в **Settings → Secrets and variables → Actions → Secrets**.
2. (Необязательно) переопредели `JOBBOT_MODEL`, `JOBBOT_TOP_N`, `JOBBOT_MIN_SCORE`
   в **Variables**.
3. Расписание (`on: schedule`) активируется только на ветке по умолчанию (`main`).
   Пока работаешь в ветке `claude/job-search-bot-pt041v` — запускай вручную через
   **Run workflow** (выбрав эту ветку) либо смёрджи в `main`.

Результаты каждого запуска бот коммитит обратно: смотри `reports/latest.md` и папку
`applications/`.

## Структура

```
config/profile.yaml      # профиль соискателя (роль, ключевые слова, локация)
config/sources.yaml      # включённые источники
resume/resume.md         # базовое резюме
jobbot/                  # код бота
  sources/               # сборщики по источникам (hh, remoteok, wwr, telegram, linkedin)
  scoring.py             # балл соответствия
  ai.py                  # Claude: письма + адаптация резюме
  pipeline.py            # оркестрация
data/seen.json           # дедуп (генерируется)
reports/                 # отчёты (генерируются)
applications/            # отклики (генерируются)
```

## Замечания по источникам

- **LinkedIn / Indeed** не дают надёжного доступа без логина/официального API и
  активно блокируют запросы с серверных IP. LinkedIn включён как best-effort;
  Indeed не реализован осознанно (Cloudflare-защита делает его нестабильным).
  Самые надёжные источники здесь — **HeadHunter**, **RemoteOK**, **We Work Remotely**
  и **Telegram**.
